CRMS — Complete Loan Lifecycle Tutorial

This step-by-step guide walks you through the entire corporate loan application process, from creation to offer letter generation. Every button click, form field, and role transition is documented with exact values to enter.

14 Steps 7 Roles Involved Mock-Service Compatible ~45 min end-to-end

Overview & WorkflowUnderstand the full lifecycle before you begin

Mock Services: This tutorial is designed to work with mock external services (Core Banking, SmartComply CAC, Credit Bureau, Fineract). Submit for ReviewLoan Officer submits the application for branch approval

👤 Role: Loan Officer — Username: loanofficer

Pre-Submission Validation

Before submitting, the system validates that you have:

  • At least one document uploaded (including a Bank Statement) ✔
  • Directors or Signatories information present ✔
  • Financial statements meeting the business-age requirement ✔

Submit the Application

  1. On the Application Detail page, look at the action bar at the top. You will see a Submit for Review button (visible only when status is Draft and you are a Loan Officer).
  2. Click Submit for Review.
  3. A modal appears showing any validation warnings. If all checks pass, you will see a green success message. If there are warnings, they will be listed (you can still proceed).
  4. In the Comments field, enter: Application complete with all required documents, financials, collateral, and guarantor. Ready for branch review.
  5. Click Confirm Submit.
  6. The status changes from Draft → BranchReview.
Status: BranchReview — The application is now in the Branch Approver's queue. Log out of the Loan Officer account. The next action must be taken by a Branch Approver.

ⓘ
Repayment Schedule Calculation: The system uses a hybrid approach. If the loan product has a FineractProductId mapped and the Fineract API is reachable, it uses the Fineract API for exact schedule calculation. Otherwise, it falls back to in-house financial math (EMI formula for reducing balance). In mock mode, the in-house calculation is used.

What Happens Next (Production)

In a production environment, after generating the offer letter:

  1. The offer letter PDF is printed and sent to the customer for signature.
  2. Once the customer accepts and signs, the Operations team manually books the loan in the core banking system (Fineract).
  3. After booking, the Operations user can update the status to Disbursed by recording the core banking loan reference number.

Automated disbursement via API is available in the system but intentionally not exposed in the UI for audit/compliance reasons.

TroubleshootingCommon issues and solutions

Cannot see "Submit for Review" button

  • The button only appears when status is Draft and you are logged in as a Loan Officer.
  • Check that you are on the correct application's Detail page.

Cannot see Approve/Reject/Return buttons

  • These buttons are role-specific:
    • BranchReview: only visible to BranchApprover
    • HOReview: only visible to CreditOfficer
    • CommitteeApproved: Approve visible to FinalApprover
  • Make sure you are logged in with the correct role for the current status.

"Setup Committee" button not showing

  • This button only appears when:
    • Status is CommitteeCirculation
    • No committee review exists yet for this application
    • You are logged in as Credit Officer

"Loan Pack" or "Offer Letter" button not visible

  • Loan Pack: visible when status is Approved, CommitteeApproved, or Disbursed
  • Offer Letter: visible when status is Approved or Disbursed
  • If you just changed the status, refresh the page.
  • If buttons still do not appear, check the browser console for errors. The most common cause is a missing database column (migration not applied).

Account number "1234567890" returns error

  • Ensure CoreBanking:UseMock is set to true in appsettings.json.
  • If using the real API, this account number will not exist in the real core banking system.

Credit checks not running after branch approval

  • Credit checks are triggered automatically by a background process after branch approval.
  • Ensure SmartComply:UseMock is set to true in appsettings.json.
  • The BVNs entered for directors/signatories/guarantors must match the mock data BVNs listed in the Mock Data Reference section.
  • Wait a few seconds after branch approval, then refresh the page and check the Bureau tab.

Balance sheet validation fails

  • Total Assets MUST exactly equal Total Liabilities + Total Equity.
  • Example: Assets=2,500,000,000 must equal Liabilities=1,200,000,000 + Equity=1,300,000,000.
  • Double-check your numbers. The system does not allow rounding differences.

Committee vote validation errors

  • Approved Amount must be positive and within a reasonable range of the requested amount.
  • Tenor and Interest Rate must be within valid ranges.
  • Minimum approval votes cannot exceed required votes in committee setup.

Application shows mock/demo data instead of real data

  • If logged in as admin, the system may show mock demonstration data. Log in as the appropriate role-specific user instead.
  • If the Detail page shows status "HOReview" when you expect "Draft", this usually means the database query is failing silently. Check that all EF Core migrations have been applied.
